开发 / 运维 / 自建服务

今天给自己的 Codex 工作流加“成本记录”

AI 编程工具从订阅制转向更细的用量计费，是一个明显趋势。别只看“它能不能写代码”，还要记录“它写一次值不值”。

小动作：给项目加一个 AI_WORKLOG.md：任务、模型、耗时、人工返工、是否合并。以后你就能知道哪个工作流真省钱。

自建服务先做低权限 Agent

如果你后面要让 AI 管 NAS、GitHub、服务器或日报仓库，别一开始就给写权限和删除权限。

建议：先做“读取 + 生成补丁 + 人工批准”，再升级到“自动写入”。日报仓库可以继续当测试场。

网页化方向正确：index.html 比长对话更适合手机

这次测试证明一个关键点：新闻内容如果最终要给手机看，HTML 首页比 Markdown 分散文件更接近最终形态。

下一步：可以给 GitHub Pages 加一个固定公开入口，再考虑加搜索和历史归档导航。

风险 / 安全 / 政策提醒

AI 写代码不是免审：安全和权限边界要前置

关于 AI 编码助手的研究和行业讨论持续指向几个风险：数据泄露、许可证问题、提示注入、不安全代码建议，以及工具调用失败。

避坑：涉及密钥、服务器、数据库、NAS 权限的代码，必须人工审查；不要让 Agent 自己处理生产密钥。

参考：arXiv - AI coding assistant security concerns

别被“AI Agent 全自动”带节奏

Google DeepMind 最近也在讨论如何借鉴网络安全思路来约束更自治的 AI Agent：越能自主行动，越要有监控、权限隔离和审计。

安安规则：任何自动化任务都要有三件东西：日志、可回滚、失败兜底。

来源：Axios

安全基本功仍然比追热点更重要

企业安全讨论仍然反复强调：很多事故不是科幻级攻击，而是身份控制、补丁、权限、备份和演练没做好。

今天可做：检查你自己的 GitHub token、NAS 共享权限、服务器密钥和自动化任务权限范围。

来源：The Times
